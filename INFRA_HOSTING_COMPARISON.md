## 1. 비교 전제 조건

### 사용자 요구사항 (확정)

| 항목 | 내용 |
|------|------|
| 초기 사용자 규모 | 사이냅 100~300명, 1개 회사 |
| 향후 확장 | 수십~수백개 회사 (SaaS 전환) |
| 데이터 규모 | 분기당 MBO+역량+다면+설문, 1인 50~100건 레코드, 첨부 5건/인 |
| 추정 연간 데이터 | 100명 기준 약 8만 건 + 첨부 5GB |
| 백업 정책 | 일간 7일 + 주간 4주 + 월간 12개월 |
| 재해 복구 | 외부 고객 서비스 단계에서 추가 검토 (초기 미적용) |
| 트래픽 패턴 | 평가 기간(월 1~2회, 약 1주) 피크 |
| PostgreSQL 버전 | 16 (LTS) |

### 가중치 (5점 만점, ×2는 두 배 가중)

| 비교 항목 | 가중치 |
|-----------|--------|
| **신뢰도·인지도** | **×2** |
| 인증 (ISMS-P/CSAP/KISA) | ×2 |
| 성능 (CPU/RAM/IOPS) | ×2 |
| 가격 (월 운영비) | ×2 |
| 국내 데이터센터 | ×1 |
| PostgreSQL 매니지드 품질 | ×1 |
| Object Storage 품질·가격 | ×1 |
| 운영 편의성 | ×1 |

**신뢰도·인지도가 가중 평가 항목인 이유**:
인사평가 데이터는 직원의 평가·승진·연봉에 직결되는 민감 정보. 이런 데이터를 다루는 시스템은 "어디서 호스팅되는가"가 고객사 신뢰의 핵심 요인. 작은 가격 차이보다 압도적 인지도·시장지위가 영업·신뢰 면에서 더 큰 가치 제공.

### 비교 대상

해외 자본 클라우드(AWS, Azure, GCP)는 의도적 제외 — 처리방침·국외이전 동의서 부담 회피, 국내 데이터 보관 원칙.

| 클라우드 | 선정 이유 |
|----------|----------|
| **NCloud (Naver Cloud Platform)** | 국내 1위 시장점유율, 인증 가장 풍부 |
| **KT Cloud** | 통신사 기반, 공공 강점 |
| **NHN Cloud (TOAST)** | 가격 경쟁력, OpenStack 기반 |
| **Kakao Cloud (Kakao Enterprise)** | 가장 공격적 가격 정책 |

## 4. 사용자 결정

### 선정: NCloud (Naver Cloud Platform)

### 선정 근거

1. **인사 데이터 민감성 고려 시 신뢰도·인지도가 핵심 평가 요소**
   - 인사평가는 직원의 평가·승진·연봉에 직결되는 민감 정보
   - 고객사 인사 담당자가 "어디서 호스팅되는가"를 신뢰 판단의 핵심 신호로 봄
   - 국내 1위 NCloud는 이 영역에서 가장 강한 신뢰 신호 제공

2. **가중 점수 1위 (54.0/60, 달성률 90%)**
   - 신뢰도·인지도 항목 추가 후 NCloud가 자연스럽게 1위로 수렴
   - 2위 NHN Cloud(52.5/60) 대비 1.5점 우위

3. **장기 안정성**
   - 국내 IaaS 시장 점유율 1위, 모회사(네이버) 재무 건전성 최고
   - 라인·네이버페이·삼성·LG·현대 등 대기업 레퍼런스 다수
   - B2B SaaS 전환 시 대기업 고객사 영업에서 강력한 신뢰 신호

4. **처음부터 최종 선택지로 시작 — 마이그레이션 비용 회피**
   - 사내 운영 → 전용 인스턴스 판매 → SaaS 전환 모든 단계에서 동일 클라우드 사용
   - 향후 클라우드 이전 비용·리스크 0

### 약점 인정 및 보완책

| 약점 | 보완책 |
|------|--------|
| NHN Cloud 대비 월 ₩72,000 가격 프리미엄 | MSP 파트너 할인(5~15%) + 1년 약정 할인(10~20%)으로 상쇄 가능 |
| 매니지드 PostgreSQL 백업 보관 최대 30일 | 별도 백업 스크립트 → Object Storage 내보내기로 일/주/월 백업 정책 구현 |
