### ISSUE-001 — 평가 기간 한정 열람 정책 옵션 점검 및 보완

**우선순위**: P1
**발견일**: 2026-05-21 (PROMPT 52)
**관련 원칙**: HRPRIVACY 원칙 4 (운영자 접근 정책 — 평가 데이터 열람 권한)
**상태**: 미해결

**내용**:
관리자가 "평가 기간 한정 열람" 정책을 설정 시, 평가 기간 종료 후에는 본인·상사·1차 평가자 모두 평가 결과를 볼 수 없어야 함. 현재 시스템에 일부 구현됐는지 또는 누락됐는지 점검 필요.

**원칙 명시 내용**:
- 본인의 모든 평가 열람 — 평가 기간 한정 옵션 가능
- 상사가 부하의 모든 평가 열람 — 평가 기간 한정 옵션 가능
- 1차 평가자의 자기 평가 내용 열람 — 평가 기간 한정 옵션 가능

**해결 요건**:
- [ ] 현재 코드에서 평가 기간별 열람 권한 체크 위치 확인
- [ ] 관리자 정책 설정 UI 확인 (이미 있는지)
- [ ] 누락된 부분 식별 + 보완 PROMPT 작성

**예상 작업량**: 중

### ISSUE-002 — 평가 기간 비활성화 시 부하·상사 열람 차단 점검

**우선순위**: P1
**발견일**: 2026-05-21 (PROMPT 52)
**관련 원칙**: HRPRIVACY 원칙 4
**상태**: 미해결

**내용**:
관리자가 특정 평가 기간을 비활성화하면, 해당 기간의 평가 결과는 다음과 같이 처리되어야 함:
- 일반 직원·상사·2차 평가자: 열람 불가
- 인사팀(master/admin, 단계 무관): 열람 가능

현재 시스템이 이 정책을 정확히 적용하는지 점검 필요.

**해결 요건**:
- [ ] `/api/eval-periods/:id/toggle` 라우터의 비활성화 동작 확인
- [ ] 평가 조회 API들의 비활성화 기간 필터링 여부 확인
- [ ] 인사팀 권한은 예외 적용되는지 확인
- [ ] 누락 부분 보완

**예상 작업량**: 소~중

### ISSUE-003 — 퇴사자 데이터 5년 자동 삭제 스케줄러 미구현

**우선순위**: P2
**발견일**: 2026-05-21 (PROMPT 52)
**관련 원칙**: HRPRIVACY 원칙 3 (떠날 자유 — 5년 보존 후 자동 삭제)
**상태**: 미해결

**내용**:
사용자(직원) 퇴사 후 5년 시점에 평가 데이터 + 첨부 파일을 cryptographically irreversible 영구 삭제하는 스케줄러 필요.

**해결 요건**:
- [ ] `users` 테이블에 퇴사일 컬럼 검토 (현재 `is_active`만 있음 → 퇴사일 별도 컬럼 필요할 수 있음)
- [ ] 5년 경과 사용자 일괄 삭제 스케줄러 (cron 또는 Node-cron)
- [ ] DB row 삭제 + Object Storage 객체 실삭제 (INFRA-2C 완료 후 통합)
- [ ] 삭제 작업 자체를 audit_log에 기록

**예상 작업량**: 중
**의존성**: INFRA-2C (Object Storage 도입) 완료 후 통합 권장

### ISSUE-004 — 본인 평가 데이터 ZIP/PDF 내보내기 기능 미구현

**우선순위**: P2
**발견일**: 2026-05-21 (PROMPT 52)
**관련 원칙**: HRPRIVACY 원칙 3 (떠날 자유 — 본인 데이터 ZIP/PDF 다운로드)
**상태**: 미해결

**내용**:
재직 중 사용자가 본인의 평가 데이터 전체(목표·KPI·자평·상사평·2차평·등급·점수 + 첨부 파일)를 ZIP 또는 PDF로 다운로드 가능해야 함. 퇴사 시점에 인사팀이 사본을 발급할 수도 있어야 함.

**해결 요건**:
- [ ] 본인 평가 데이터 export API 신규 작성
- [ ] ZIP/PDF 둘 다 지원할지, 한쪽만 지원할지 결정
- [ ] 첨부 파일도 포함할지 결정 (포함하면 ZIP 권장)
- [ ] 인사팀 대행 발급 기능 (퇴사자용)

**예상 작업량**: 중
